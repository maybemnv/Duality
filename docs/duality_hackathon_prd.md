# PRD — Duality AI Offroad Semantic Scene Segmentation
### Finalized Model: SegFormer-B2 | Hardware: RTX 3070 Laptop 8GB VRAM

---

## 1. Problem Statement

Train a semantic segmentation model on synthetic desert imagery generated by Duality AI's Falcon platform. The model must correctly classify every pixel across 10 environmental classes and generalize to a **completely different desert location** it has never seen during training.

**Primary metric:** Mean IoU (80 pts)  
**Secondary metric:** Report clarity (20 pts)  
**Core challenge:** Domain shift — train and test environments are different desert locations

---

## 2. Finalized Model

### SegFormer-B2

| Property | Value |
|----------|-------|
| Architecture | Mix Transformer (MiT-B2) encoder + lightweight MLP decode head |
| Pretrained on | ImageNet-1K + ADE20K semantic segmentation |
| Source | HuggingFace `transformers` |
| Model string | `nvidia/segformer-b2-finetuned-ade-512-512` |
| Parameters | ~25M |
| Output | Per-pixel logits → 10 class predictions |

### Why SegFormer-B2 (decision rationale)

**1. Dense prediction requirement**  
Segmentation outputs one class label per pixel. For a 512×512 image that is 262,144 predictions per forward pass. This rules out LLMs, SLMs, and image classifiers — none produce pixel-level masks natively. Only encoder-decoder vision architectures are suitable.

**2. Pretrained weights over training from scratch**  
The dataset is synthetic and hackathon-sized. A randomly initialized model needs far more data to learn low-level features (edges, textures, shapes) that a pretrained backbone already knows from ImageNet. Fine-tuning converges in hours. Training from scratch takes days and produces worse IoU.

**3. Transformer over CNN for domain shift**  
The test environment is a different desert location from training. CNNs learn local spatial patterns via small convolution kernels — if rock color or grass texture shifts between environments, local patterns break down. SegFormer uses self-attention: every pixel attends to every other pixel globally, learning semantic context ("brown cluster surrounded by dry vegetation in open terrain = Dry Bushes") rather than location-specific textures. This transfers across environments significantly better than CNN-based approaches.

**4. ADE20K pretraining relevance**  
ADE20K contains outdoor scenes, vegetation, terrain, sky — directly relevant to desert segmentation. SegFormer's pretrained weights already encode what bushes, rocks, and ground look like at a feature level.

**5. Hardware fit**  
SegFormer-B2 uses ~5.5–6GB VRAM at batch size 8 with 512×512 images and AMP enabled. RTX 3070 Laptop (8GB VRAM) handles this with headroom.

```
AMP          : True            (mixed precision — saves ~1.5GB VRAM)
PIN_MEMORY   : True            (faster CPU to GPU transfer)
```

### Mixed Precision (AMP) — mandatory on laptop GPU

AMP runs the forward pass in float16 instead of float32. This reduces VRAM usage by 30–40% with no accuracy loss. On a laptop GPU with power constraints, this is not optional — enable it every run.

### Laptop throttling checklist

Run these before every training session:

- Plug into power (never train on battery)
- Set Windows power plan to High Performance
- Use a cooling pad or laptop stand
- Monitor with `nvidia-smi dmon` — keep GPU temp under 85°C
- Keep GPU utilization above 90% — if it drops, reduce `num_workers`

### Segmentation classes

| Class ID | Class Name | Remapped ID | Expected difficulty |
|----------|------------|-------------|---------------------|
| 100 | Trees | 0 | Easy — tall distinctive silhouette |
| 200 | Lush Bushes | 1 | Medium — confused with Dry Bushes |
| 300 | Dry Grass | 2 | Hard — blends with Landscape |
| 500 | Dry Bushes | 3 | Hard — similar color to Rocks |
| 550 | Ground Clutter | 4 | Hard — very low pixel count |
| 600 | Flowers | 5 | Hard — rare class, imbalance risk |
| 700 | Logs | 6 | Hardest — occlusion, rarest class |
| 800 | Rocks | 7 | Hard — texture similar to ground |
| 7100 | Landscape | 8 | Easy pixels but causes imbalance |
| 10000 | Sky | 9 | Easiest — clear distinct boundary |

### Critical: Class ID remapping

PyTorch CrossEntropyLoss requires class labels in range `[0, N-1]`. The provided mask IDs are non-sequential (100, 200 ... 10000). Remapping is mandatory — training will silently produce garbage outputs without it.

```
Original → Remapped
100      → 0   (Trees)
200      → 1   (Lush Bushes)
300      → 2   (Dry Grass)
500      → 3   (Dry Bushes)
550      → 4   (Ground Clutter)
600      → 5   (Flowers)
700      → 6   (Logs)
800      → 7   (Rocks)
7100     → 8   (Landscape)
10000    → 9   (Sky)
```

Verification step: after applying remapping, visualize 5 random masks and confirm labels match visible class boundaries before starting any training run.

### Environment setup

```
1. Install Miniconda or Anaconda
2. Open Anaconda Prompt
3. Navigate to ENV_SETUP folder
4. Run: setup_env.bat              (Windows)
         or create setup_env.sh    (Mac / Linux equivalent)
5. This creates conda env: EDU
6. Activate: conda activate EDU
7. Install additional deps:
   pip install transformers accelerate albumentations
```

### Training workflow

```
Step 1: Run sample train.py as-is
        → Record baseline IoU (expected: 0.25–0.35)
        → This is your comparison baseline

Step 2: Apply class ID remapping in Dataset class
        → Visualize 5 masks to verify correctness

Step 3: Load SegFormer-B2 with swapped decode head (10 classes)
        → Freeze backbone for first 5 epochs
        → Then unfreeze for full fine-tuning

Step 4: Compute class weights from training pixel frequencies
        → Apply as weights in CrossEntropyLoss

Step 5: Add augmentation pipeline (see Section 6)

Step 6: Train 60 epochs with AMP enabled
        → Save checkpoint every 5 epochs
        → Keep best checkpoint by val IoU

Step 7: Run test.py on testImages
        → Record final IoU per class + mean IoU
        → These are your submission numbers
```

### Differential learning rate — why it matters

The backbone (MiT-B2 encoder) already has good feature representations from ADE20K pretraining. Applying a high learning rate destroys those pretrained weights. The decode head is randomly initialized and needs a higher rate to learn quickly.

```
Backbone LR  : 6e-6   (preserve pretrained features)
Decode head  : 6e-5   (learn quickly from scratch)
Ratio        : 10x difference
```

### Training loop structure

```
For each epoch:
  For each batch:
    1. Load image + remapped mask → GPU
    2. Forward pass (float16 with AMP)
    3. Compute weighted CrossEntropyLoss
    4. Backward pass with gradient scaling
    5. Update optimizer
    6. Update LR scheduler

  After each epoch:
    1. Compute val loss
    2. Compute per-class IoU on val set
    3. Save checkpoint if mean IoU improved
    4. Log metrics to runs/ directory

Early stopping: if val loss does not improve for 10 epochs, stop
```

## 6. Augmentation Strategy

Augmentation is the primary tool for improving generalization across the domain shift between train and test environments. Apply all transformations to both the RGB image and its corresponding mask simultaneously.

### Must-have augmentations

| Augmentation | Parameters | Reason |
|-------------|------------|--------|
| RandomResizedCrop | scale=(0.5, 1.0), size=512 | Simulates different camera distances, handles multi-scale objects |
| HorizontalFlip | p=0.5 | Free augmentation, zero cost |
| ColorJitter | brightness=0.3, contrast=0.3, hue=0.1 | Desert lighting shifts heavily by time of day and sun angle |
| Normalize | mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225] | Required for ImageNet-pretrained backbone |

### Recommended augmentations (add after baseline)

| Augmentation | Parameters | Reason |
|-------------|------------|--------|
| GaussianBlur | blur_limit=(3, 7), p=0.3 | Synthetic images are too sharp — blur bridges sim-to-real gap |
| RandomGrayscale | p=0.1 | Prevents overreliance on color cues |
| RandomRotation | degrees=10 | Desert terrain looks similar at slight angles |
| GridDistortion | distort_limit=0.2, p=0.3 | Mimics lens distortion, improves boundary robustness |

### For rare classes (Logs, Flowers, Ground Clutter)

| Technique | Reason |
|-----------|--------|
| CoarseDropout (CutOut) | Simulates occlusion — Logs are frequently hidden behind vegetation |
| Dataset oversampling | Weight the sampler to show rare-class images more frequently |
| Copy-paste augmentation | Copy rare class instances into other training images |

### Validation augmentation

Validation set uses only Resize + Normalize. No random transforms — validation must be deterministic for consistent IoU tracking.

## 7. Loss Function and Class Imbalance

### The imbalance problem

Landscape (class 8) and Sky (class 9) will dominate pixel counts across the dataset. Logs (class 6), Flowers (class 5), and Ground Clutter (class 4) will appear in very few pixels. A naive CrossEntropyLoss optimizes for the majority classes and effectively ignores the rare ones, producing IoU = 0 for Logs and Flowers.

Since mean IoU averages all 10 classes equally, a model that gets 0 IoU on even one class is penalized heavily.

### Solution: weighted CrossEntropyLoss

```
1. Count pixel frequency per class across entire training set
2. Invert frequencies: rare class → high weight, dominant class → low weight
3. Normalize weights so they sum to num_classes
4. Pass weights tensor to CrossEntropyLoss
```

Computed weight formula:
```
weight[c] = 1 / (pixel_count[c] + epsilon)
weight    = weight / weight.sum() * num_classes
```

Epsilon (1e-6) prevents division by zero for classes with very few pixels.

## 8. Evaluation

### Primary metric: Mean IoU

```
IoU per class = True Positives / (True Positives + False Positives + False Negatives)
Mean IoU      = average of IoU across all 10 classes
```

Every class contributes equally to the mean. A class with IoU = 0 has the same drag on the score as a class with IoU = 0.5.

### Metrics to log every epoch

| Metric | Purpose |
|--------|---------|
| Training loss | Track learning — should decrease steadily |
| Validation loss | Detect overfitting — should decrease then plateau |
| Per-class IoU (all 10) | Identify which classes are underperforming |
| Mean IoU | Primary score — save checkpoint on improvement |
| Inference time (ms) | Target < 50ms per image |

### Diagnosing training issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| Val loss increases while train loss decreases | Overfitting | Add more augmentation, reduce LR |
| Loss plateaus too high after many epochs | Underfitting | Increase epochs, unfreeze backbone |
| Specific class IoU = 0 | Class remapping bug or severe imbalance | Verify remapping, increase class weight |
| GPU utilization below 80% | DataLoader bottleneck | Increase num_workers, enable pin_memory |
| CUDA OOM error | Batch too large for VRAM | Reduce batch_size to 4, verify AMP is on |

### IoU benchmark targets

| Stage | Expected Mean IoU |
|-------|-------------------|
| Baseline (sample train.py) | 0.25 – 0.35 |
| SegFormer-B2, no augmentation | 0.45 – 0.55 |
| SegFormer-B2, full augmentation + weighted loss | 0.55 – 0.70 |
| With TTA (test-time augmentation) | +0.02 – 0.05 gain |

## 9. Optimization Techniques (post-baseline)

Apply these in order. Measure IoU delta after each one before adding the next.

### 1. Test-Time Augmentation (TTA)

During inference on testImages, run each image through the model multiple times with different augmentations and average the predictions. This consistently adds 2–5 IoU points with no additional training.

```
Augmentations to average at test time:
- Original image
- Horizontally flipped image (flip prediction back)
- Slightly scaled image (0.9x, 1.1x)
Average the softmax outputs, then argmax
```

### 2. Cosine Annealing LR scheduler

```
Start LR   : 6e-5
Min LR     : 1e-6
T_max      : 60 (number of epochs)
Effect     : LR decreases smoothly, avoids sharp drops
```

### 3. Gradient clipping

Prevent exploding gradients during fine-tuning:
```
Max gradient norm: 1.0
Apply before optimizer.step()
```

### 4. Checkpoint ensembling (if time permits)

Save the top 3 checkpoints by val IoU during training. At inference, average their predictions. Usually adds 1–3 IoU points at zero additional training cost.

### Failure case documentation format (pages 5–6)

For each failure case, document in this exact format:

```
Task:    [what the model was predicting]
Issue:   [what went wrong — include a prediction image]
Why:     [root cause analysis]
Fix:     [what was changed]
Result:  [IoU before → IoU after the fix]
```

Minimum failure cases to document: class imbalance (Logs/Flowers), domain shift behavior, any misclassification pattern between visually similar classes (Dry Bushes vs Rocks, Dry Grass vs Landscape).

### Report scoring strategy

The report is worth 20 points. These are essentially free points if the process is documented. Judges reward:

- Showing an iteration loop (table of IoU scores across experiments)
- Honest failure analysis with images
- Explaining reasoning, not just results
- Clear reproducible methodology
